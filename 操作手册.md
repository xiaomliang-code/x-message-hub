# 发布 x-message-hub 完整操作手册

> 按顺序执行，每一步都有具体命令，复制粘贴即可。

---

## 准备工作（只做一次）

### 1. 注册账号

需要注册两个账号：

| 平台 | 地址 | 用途 |
|------|------|------|
| GitHub | https://github.com | 存放代码 |
| PyPI | https://pypi.org/account/register/ | 发布安装包 |
| TestPyPI | https://test.pypi.org/account/register/ | 发布前测试用（可选） |

> **建议：** GitHub 和 PyPI 用同一个邮箱注册，方便管理。

### 2. 安装必要工具

在终端执行：

```bash
pip install build twine
```

## 第一阶段：整理本地文件

### 步骤 1：新建项目文件夹

```bash
# 在你喜欢的位置新建文件夹（比如桌面或 Developer 目录）
mkdir ~/Developer/x-message-hub
cd ~/Developer/x-message-hub
```

### 步骤 2：创建目录结构

```bash
mkdir -p src/x_message_hub
mkdir -p examples
mkdir -p tests
```

### 步骤 3：复制代码文件

把以下文件放到对应位置：

```
x-message-hub/
├── src/
│   └── x_message_hub/
│       ├── __init__.py          ← 从下载的文件里找
│       └── message_hub.py       ← 核心库
├── examples/
│   └── demo.py                  ← 示例文件（message_hub_demo.py 改名）
├── tests/
│   ├── __init__.py              ← 空文件
│   └── test_basic.py            ← 测试文件
├── README.md                    ← 英文文档
├── README_CN.md                 ← 中文文档
├── LICENSE                      ← MIT 协议
├── pyproject.toml               ← 打包配置
└── .gitignore                   ← Git 忽略规则
```

### 步骤 4：验证文件是否正确

```bash
# 确认结构
ls -la
ls -la src/x_message_hub/

# 确认可以正常导入
cd ~/Developer/x-message-hub
python3 -c "
import sys
sys.path.insert(0, 'src')
from x_message_hub import MessageHub
print('导入成功，版本:', MessageHub.__module__)
"
```

### 步骤 5：运行测试，确保代码没问题

```bash
cd ~/Developer/x-message-hub
pip install pytest
pytest tests/ -v
```

看到所有测试 PASSED 才继续。

## 第二阶段：上传到 GitHub

### 步骤 6：在 GitHub 创建仓库

1. 打开 https://github.com，点右上角 **`+`** → **New repository**
2. 填写：
   - **Repository name**：`x-message-hub`
   - **Description**：`Production-grade thread-safe message bus for Python`
   - 选择 **Public**（公开）
   - **不要**勾选任何初始化选项（README、.gitignore、License 都不勾）
3. 点 **Create repository**
4. 页面会显示你的仓库地址，类似：
   ```
   https://github.com/你的用户名/x-message-hub.git
   ```
   **记下这个地址，下一步要用。**

### 步骤 7：初始化 Git 并上传

```bash
cd ~/Developer/x-message-hub

# 初始化 git
git init

# 设置你的 git 用户信息（只需设置一次）
git config --global user.email "你的邮箱@example.com"
git config --global user.name "Jason"

# 添加所有文件
git add .

# 第一次提交
git commit -m "Initial release: x-message-hub v3.0.0"

# 关联到 GitHub 仓库（替换成你的用户名）
git remote add origin https://github.com/你的用户名/x-message-hub.git

# 设置主分支名称
git branch -M main

# 上传
git push -u origin main
```

上传成功后，打开 `https://github.com/你的用户名/x-message-hub`，就能看到代码了。

### 步骤 8：打上版本标签（Release）

```bash
cd ~/Developer/x-message-hub

# 创建 v3.0.0 标签
git tag -a v3.0.0 -m "x-message-hub v3.0.0 - Initial Release"

# 把标签推送到 GitHub
git push origin v3.0.0
```

然后在 GitHub 页面：
1. 点右侧 **Releases** → **Create a new release**
2. Tag 选 `v3.0.0`
3. Title 填：`v3.0.0 - Initial Release`
4. 描述填：
   ```
   First public release of x-message-hub.

   Features:
   - Three-tier pipeline architecture
   - Dynamic QoS auto-demotion
   - Zero-copy numpy array support
   - Request-Reply pattern helper
   - No required dependencies (Python 3.9+)
   ```
5. 点 **Publish release**

## 第三阶段：发布到 PyPI

### 步骤 9：修改 pyproject.toml 里的 GitHub 地址

用文本编辑器打开 `pyproject.toml`，把：
```
https://github.com/your-username/x-message-hub
```
替换成你真实的地址：
```
https://github.com/你的用户名/x-message-hub
```

### 步骤 10：构建安装包

```bash
cd ~/Developer/x-message-hub

# 构建（会生成 dist/ 文件夹）
python3 -m build

# 确认生成了两个文件
ls dist/
# 应该看到：
# x_message_hub-3.0.0-py3-none-any.whl
# x_message_hub-3.0.0.tar.gz
```

### 步骤 11：先在 TestPyPI 测试（推荐）

TestPyPI 是 PyPI 的测试环境，发布在这里不影响真实的 PyPI。

```bash
# 上传到 TestPyPI
twine upload --repository testpypi dist/*
```

会提示输入用户名和密码，填你注册 TestPyPI 的账号。

上传成功后，用测试地址安装验证：
```bash
pip install --index-url https://test.pypi.org/simple/ x-message-hub
python3 -c "from x_message_hub import MessageHub; print('安装成功')"
```

### 步骤 12：正式发布到 PyPI

测试没问题后，发布到正式 PyPI：

```bash
twine upload dist/*
```

填你注册 PyPI 的账号和密码。

上传成功后，等约 1 分钟，任何人都可以：
```bash
pip install x-message-hub
```

## 第四阶段：验证发布成功

### 步骤 13：验证安装

```bash
# 新开一个终端，或者先卸载本地版本
pip uninstall x-message-hub -y

# 从 PyPI 安装
pip install x-message-hub

# 验证
python3 -c "
from x_message_hub import MessageHub
import time

bus = MessageHub()
received = []
bus.subscribe(['test'], lambda ch, m: received.append(m))
bus.send_message('test', {'hello': 'world'})
time.sleep(0.3)
bus.stop()
print('✅ 发布成功！收到消息:', received[0])
"
```

## 之后每次更新代码的流程

```bash
cd ~/Developer/x-message-hub

# 1. 修改代码...

# 2. 更新版本号（pyproject.toml 里的 version）
#    比如从 3.0.0 → 3.0.1

# 3. 提交到 GitHub
git add .
git commit -m "修复了某某问题 / 新增了某某功能"
git push

# 4. 打新标签
git tag -a v3.0.1 -m "v3.0.1 - 修复了某某问题"
git push origin v3.0.1

# 5. 重新构建并发布
rm -rf dist/
python3 -m build
twine upload dist/*
```

## 遇到问题

**问题：push 时提示需要输入密码**
```bash
# GitHub 现在用 Token 代替密码
# 去 GitHub → Settings → Developer settings → Personal access tokens → Tokens (classic)
# 生成一个 Token，勾选 repo 权限
# push 时密码那里填这个 Token
```

**问题：twine upload 失败，提示包名已存在**
```bash
# 说明 PyPI 上已有同名包，需要换个包名
# 修改 pyproject.toml 里的 name 字段，重新 build
```

**问题：pip install 安装后 import 报错**
```bash
# 检查包名和模块名的区别：
pip install x-message-hub        # 包名：用连字符
from x_message_hub import ...    # 模块名：用下划线
```
